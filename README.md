# Six Commits from Linus

Discover how a public GitHub contribution trail connects you to notable open-source developers.

**Live demo:** https://poliklot.github.io/six-commits-from-linus/

Six Commits from Linus is a static GitHub Pages app for exploring public repository contributor paths. It is intentionally lightweight: no OAuth, no backend, no database, no tracking, and no private GitHub data.

## What changed in v2

The first MVP only searched a small target list. The current version is built for a stronger demo and GitHub Developer Program submission:

- **Power Mode:** add a repo you contributed to, for example `prettier/prettier`.
- **Closest mode:** find the closest notable developers instead of picking one target first.
- **Expanded graph:** 85 notable developers, 98 bridge repositories, and 150+ indexed public repositories.
- **Prettier case:** includes `prettier/prettier` and Prettier ecosystem targets such as `@vjeux`, `@fisker`, `@sosukesuzuki`, and `@jlongster`.
- **Static graph:** generated by GitHub Actions with the built-in `GITHUB_TOKEN` and served as JSON from GitHub Pages.

## How it works

The app indexes public contributors from high-signal open-source repositories:

```text
GitHub Actions
  -> GitHub REST API contributors endpoint
  -> public/data/famous-index.json
  -> static frontend on GitHub Pages
```

A connection exists when two users appear in the same public repository contributor graph.

Example:

```text
@you
  -> prettier/prettier
@vjeux
```

Or through a bridge contributor:

```text
@you
  -> your/project
@middledev
  -> facebook/react
@gaearon
```

## Why repo hints matter

GitHub does not provide a perfect public endpoint for “all repositories this user has ever contributed to.” Without OAuth, the app can only do a best-effort username scan.

Repo hints make the graph much sharper:

```text
GitHub username: your-login
Repo you contributed to: prettier/prettier
```

If GitHub does not return your username in the checked contributor window, the app labels the result as a repo-hint connection rather than proof.

## Local setup

```bash
npm install
cp .env.example .env
npm run build:index
npm run dev
```

For best local graph generation, put a token in `.env`:

```text
GITHUB_TOKEN=ghp_xxx
```

The token is only used by Node.js scripts. It is never bundled into frontend code.

## Scripts

```bash
npm run dev              # start Vite
npm run build            # type-check and build
npm run preview          # preview production build
npm run build:index      # build public/data/famous-index.json
npm run validate:famous  # validate target users and anchor repos
npm run lint             # ESLint
```

## GitHub Pages deployment

The project uses `.github/workflows/deploy.yml`:

1. install dependencies;
2. generate the graph with Actions `GITHUB_TOKEN`;
3. build the Vite app;
4. upload the Pages artifact;
5. deploy to GitHub Pages.

The live URL is:

```text
https://poliklot.github.io/six-commits-from-linus/
```

## Refreshing the graph

`.github/workflows/refresh-famous-index.yml` can be run manually or weekly. It runs `npm run build:index` and commits an updated `public/data/famous-index.json` if the graph changed.

## Privacy

This app only uses public GitHub data.

- It does not store usernames.
- It does not access private repositories.
- It does not require GitHub login.
- It has no backend server.
- It has no database.
- It has no analytics or tracking.

Support: support@poliklot.com

## Token safety

Never expose a GitHub token in frontend code.

Safe places for `GITHUB_TOKEN`:

- local `.env` used by Node.js scripts;
- GitHub Actions environment variables;
- private CI secrets.

Unsafe places:

- `src/**` frontend code;
- `public/**` files;
- committed `.env` files;
- screenshots, logs, or README snippets with real token values.

## Limitations

This is an approximation. It uses public repository contributor data and does not use:

- private repositories;
- OAuth-authenticated user data;
- pull request reviews;
- issues;
- stars;
- followers;
- full GitHub graph search.

No path found inside this graph does **not** mean no real GitHub connection exists.
